## Description

[Nest](https://github.com/nestjs/nest) project starter repository with Husky hooks, environment configuration, and custom setup steps.

## Getting Started

Follow these steps to set up and run the project locally:

### 1. Clone the repository

```bash
git clone https://github.com/techno-stupid/hris
```

### 2. Navigate to the project folder

```bash
cd <project-folder-name>
```

### 3. Install dependencies

```bash
yarn install
```

### 4. Set up Husky hooks

```bash
yarn prepare
```

### 5. Make hook files executable

```bash
chmod +x ./.husky/commit-msg && chmod +x ./.husky/pre-commit
```

### 6. Configure environment variables

```bash
cp .env.example .env
```

### 7. Start the development server

```bash
yarn start:dev
```

## Compile and run the project

```bash
# development
$ yarn start
# or
$ npm run start

# watch mode
$ yarn start:dev
# or
$ npm run start:dev

# production mode
$ yarn start:prod
# or
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ yarn test
# or
$ npm run test

# e2e tests
$ yarn test:e2e
# or
$ npm run test:e2e

# test coverage
$ yarn test:cov
# or
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ yarn add -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
